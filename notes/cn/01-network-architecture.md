> **考纲：概念、组成与功能、分类、性能指标、分层结构、协议、接口、服务、ISO/OSI参考模型、TCP/IP模型**

## 基本概念：
{: #section-1 }

网络由节点和链路组成，节点可以是计算机、集线器、交换机、路由器。

互连网（internet）任意通信协议，是网络的网络，通过路由器相连；因特网/互联网（Internet）TCP/IP协议族。

计算机网络是互联、自治的计算机的集合。

## 组成：
{: #section-2 }

**从组成成分看：**

由硬件（主机、通信链路、交换设备、通信处理机（网卡））、软件（FTP、EMAIL程序）、协议三大部分。

**从工作方式看：**

**分为边缘部分（服务器、个人主机、端系统：通信、资源共享）和核心部分（大量网络和路由器：提供连通性和交换）**

**从功能组成看：**

由通信子网和资源子网构成。通信子网由传输介质、通信设备、网络协议组成，使得网络具有传输、交换、控制、存储的能力，实现计算机之间的数据通信。资源子网是实现资源共享功能的设备及其软件的集合，向用户提供其他计算机的硬件、软件、数据资源的服务。

## 功能：
{: #section-3 }

数据通信、资源共享、分布式处理、提高可靠性、负载均衡

### 电路交换：

连接建立、数据传输、连接释放。通信双方独占一条物理通信链路，连接建立后以比特流直通方式收发数据，不存在存储转发，不需要分组。最初是双绞线在交换机上的交换，后来引入多路复用技术，在传输媒体上的某一频率、时隙、码序交换。

**优点：**

通信时延小，专用链路；有序传输，不存在失序；没有冲突，不会争用信道；适用范围广，适合模拟和数字信号；实时性强，连接建立随时通信；控制简单，交换设备控制容易。

**缺点：**

建立连接时间长；线路利用率低，数据传送是突发式的；灵活性差，故障发生需要重新连接；难以规格化，不同类型、速率、规格的终端很难通信；难以实现差错控制，中间结点不具备存储和检错纠错。

### 报文交换：

数据交换单位是报文，用户数据加上源地址、目的地址后分装成报文。采用存储转发技术，通过传送到相邻结点查找转发表，直至目的结点。

**优点：**

无需建立连接，没有连接时延；动态分配线路；线路可靠性高，故障后重新选择路径；线路利用率高，只占用特定链路的通信资源。提供多目标服务，同时发送给多个目的地址。

**缺点：**

转发时延高，需要将报文整体接受才能转发；缓存开销大，报文大小没有限制；错误处理效率低，报文较长，错误概率大，重传代价大。

### 分组交换：（数据报+虚电路两种）

采用存储转发技术，源结点发送前，先将报文分成等长数据段，并加上控制信息组成首部，构成分组。分组交换机通过提取首部的目的地址查找转发表，将分组传给下一个结点。

**优点：**

无需建立连接；线路利用率高，动态分配带宽，只占用分组当前传输的链路；存储管理容易，缓冲区大小固定；传输加速，分组的存储可以和前一个分组的转发以流水线方式并行工作，等待分组的时间也少；减少了出错概率和重传数据量，提高可靠性。

**缺点：**

存储转发时延仍然存在；需要额外传输控制信息，增加处理时延；数据报服务会出现失序、丢失、重复分组的问题；虚电路服务需要三个过程，增加开销。

## 分类：
{: #section-4 }

### 分布范围：

**广域网（WAN）：**

几十~几千千米，因特网的核心部分，其任务是为核心路由器提供远距离高速连接，互连分布在不同区域的城域网和局域网。

**城域网（MAN）：**

5~50km，作为城市骨干网，互连大量企业、机构和校园局域网。

**局域网（LAN）：**

几十~几千米，一般用微型计算机或工作站通过高速通信线路相连。

**个人区域网（PAN）：**

WIFI热点、蓝牙（无线技术）。

### 传输技术：

**广播式网络：**

共享信道，所有计算机（包括自己）都会接收到分组，但只有目的主机才会接受。局域网都采用、广域网中无线、卫星通信网络也采用。

**点对点网络：**

物理线路连接各个通信主机，分组交换经过中间结点存储转发。

### 拓扑结构：

计算机网络拓扑结构主要取决于通信子网。

**总线型：**

单根传输连接计算机。优点：建网容易、增删结点容易、节省线路。缺点：通信效率低、故障敏感。

**星形：**

每个终端都与中央设备（交换机/路由器）相联。优点：便于集中控制管理。缺点：成本高、中央设备故障敏感。

**环形：**

令牌环局域网。可以是单环，双环，环中信号是单向的。

**网状：**

每个结点至少两条路径和其他结点相联，多用于广域网。优点：可靠性高。缺点：控制复杂、成本高。

### 使用者：

### 公用网：因特网、电信公司出资的大型网络等。

专用网：部分特殊业务需要，不对外提供服务。

### 传输介质：

有线网络：双绞线、同轴电缆

无线网络：蓝牙、微波、无线电

### 交换技术：

电路交换、分组交换、报文交换。

## 性能指标：
{: #section-5 }

**速率（数据传输率、比特率）：**

![]({{ '/assets/notes/cn/image001.png' | relative_url }})
速率=min{主机接口速率、链路带宽、交换机或路由器接口速率}

**带宽：**

在模拟信号系统中是信号的频率范围，单位Hz赫兹；在计算机网络中是通信线路传输数据的能力，是最高数据率，单位和速率一样。

**吞吐量：**

单位时间通过某个网络或接口的数据量，包含下载和上传速率，同时也受网络带宽的限制。

**时延：**

发送时延、传播时延、处理时延、排队时延。

发送（传输）时延=分组长度（b）/发送速率（b/s）——发送时延个数=分组数+路由器节点个数

传播时延=信道长度（m）/信道传播速率（m/s）——传播时延个数=链路段数（路由器节点个数+1）

处理时延：查表转发、差错处理、提取数据时间 ——处理转发时延个数=路由器节点个数

排队时延：路由器中等待处理的时间。

![]({{ '/assets/notes/cn/image002.png' | relative_url }})

**时延带宽积：**

传播时延（s）和带宽（b/s）的乘积（以比特为单位的链路长度），可理解为在链路上的比特总量。

时延带宽积=传播时延\*带宽；最短帧长=传播时延\*2\*带宽，因此时延带宽积=2\*最短帧长。

**往返时间（RTT）：**

从发送端发送数据分组开始，到发送端收到接收端发来的相应确认分组为止，总共耗费的时间。

**信道利用率：**

信道百分之多少的时间是有数据通过的。

## 分层结构：
{: #section-6 }

### 计算机网络各层及其协议的集合称为网络的体系结构。

### 实体：

在分层结构中，第n层中活动元素称为第n层实体。实体代表任何可收发信息的硬件或软件进程。

**对等层：**

不同机器上的同一层称为对等层，同一层的实体称为对等实体。

### 协议：

对等实体间通信的规则的集合，由语法、语义、同步三部分组成。

**语法：**

数据和控制信息的格式。例如TCP报文段的格式的定义。

**语义：**

需要发出何种控制信息、完成何种动作、做出何种应答。例如TCP连接的三次握手时执行的操作定义。

**同步（时序）：**

执行各种操作的条件、时序关系。例如TCP连接的三次握手操作的时序关系定义。

### 服务：

第n层的实体不仅要使用下层的服务实现自身功能，还要向上层提供服务，该服务是n层服务的总和。

最底层只提供服务，最高层面向用户提供服务。上层只能通过相邻接口访问下层的服务。

协议的实现才能保证向上的服务，用户只能看见服务，不能看见协议。只有被看得见的功能才能称为服务。

**服务原语：**

上层使用下层的服务时，需要与下层交换一些命令，称为服务原语。

OSI参考模型将其分为四类：（用户）请求、（服务提供者）指示、（用户）响应（服务提供者的指示）、（服务提供者）证实（用户的请求）。其中有应答服务包括四类原语，无应答只有请求和指示。

**协议数据单元PDU：**

对等层之间传送数据的单元。每层PDU都分为数据和控制信息两部分，PDU=PCI+SDU。

物理层的PDU称为比特流；数据链路层：帧；网络层：分组/IP数据报；传输层：报文段/用户数据报；应用层：应用报文。无论哪一层的数据单元都可以用分组笼统表示。

**服务数据单元（SDU）：**

相邻层间交换的数据单元，SDU为服务原语的表现形式。(n)PDU=(n-1)SDU.

**接口数据单元（IDU）：**

相邻层接口上体现的数据单元，IDU=SDU（可以是上层的PDU向下传递，也可以是下层的SDU向上传递）+接口控制信息ICI。ICI是通过接口的必要信息，当越过接口后，ICI被丢弃。

**PDU封装/解封：**

`发送端第``n``层将``(n)PDU``传给第``n-1``层，第``n-1``层从``IDU``中分离出``(n-1)SDU``，加上``(n-1)PCI``组成``(n-1)PDU``，继续向下传递。接收方第``n-1``层则将收到的``PDU``解封，去除``PCI``，还原成``SDU``给第``n``层，向上传递。`

**PDU分割/组合：**

发送方第n层将一个(n)PDU分割成多个(n-1)SDU，并行从多个n-1通道发出去；接收端第n层将收到的多个SDU合并为一个(n)PDU。

**SDU分段/装配：**

发送端，当第n-1层带宽不足以传递该SDU时，第n-1层将SDU分为若干段，分别封装成多个PDU向下传递。接收方第n-1层再将收到的多个PDU解封后组装成SDU，向上传递。

**SDU拼接/分离：**

为了提高信道利用率，发送端第n-1层将多个短的SDU拼接封装为一个PDU向下传递。接收方第n-1层将PDU解封后，分离出多个SDU向上传递。

**服务分类：**

**面向连接和无连接服务：**

先建立连接，预先分配资源，如TCP。

尽最大努力交付，如IP、UDP。

**可靠和不可靠服务：**

纠错、检错、应答，尽量让数据正确可靠的到达目的地。

正确性、可靠性由应用和用户保障，让用户将错误信息报告给发送者。

**有应答和无应答服务：**

FTP，由传输系统内部自动应答。

WWW，应答由高层实现，客户端收到服务器发送的页面不给出应答。

### 接口：

相邻两层实体交换信息的逻辑接口称为服务访问点（SAP），是下层为上层提供服务的地方。

物理层：物理接口；数据链路层SAP：帧的类型字段；网络层：IP数据报的协议字段；传输层：端口号字段。

## ISO/OSI参考模型：
{: #section-7 }

低三层统称为通信子网，完成数据传输，高三层统称为资源子网，完成数据处理。

**物理层：**

规定了通信链路与通信结点的电路接口的参数，比如笔记本电脑上的网线接口。还规定了通信链路上传输信号的意义和电气特征。物理介质（双绞线、光纤、无线信道）这些不属于物理层协议之内。

**数据链路层：**

差错检测、流量控制、访问/接入控制。

**网络层：**

流量控制、差错控制、拥塞控制、网际互连。为主机提供逻辑通信。

**传输层：**

为端到端连接提供差错控制、流量控制、数据传输、服务质量。为进程提供逻辑通信。

**会话层：**

为表示层实体或用户进程建立连接，并在连接上有序传输数据，即会话，也称为建立同步。通过检查点机制维持会话，即通信失效时，从检查点恢复通信，即断点下载。可以提供会话加密，如ssh。

**表示层：**

用抽象标准方法定义数据结构，采用标准编码方式，使不同通信系统交换信息。数据压缩、加密解密。

**应用层：**

用户与网络的接口。

## TCP/IP参考模型：
{: #section-8 }

### 网络接口层：

### 表少与物理网络的接口，作用是从主机或结点接收IP分组，发到指定物理网络上。物理网络可以是局域网（以太网、令牌环网、令牌总线网）、电话网、SDH、X.25、帧中继、ATM等公共数据网。

### 网际层：

### 不保证分组有序到达，有序可靠交付由高层负责。（没有流量控制）

### 传输层：

### TCP可靠交付，数据单位是报文段；UDP尽最大努力交付，数据单位是用户数据报。

### 应用层：

### 虚拟终端协议Telnet。

![]({{ '/assets/notes/cn/image006.png' | relative_url }})

OSI在网络层支持无连接和面向连接的通信，在传输层仅有面向连接的通信。TCP/IP认为可靠性是端到端的问题，在网际层只有无连接的IP，在传输层支持无连接和面向连接的模式。

网络层负责重组分片、传输层负责重排分组。
