## Objects
{: #objects }

Objects are useful when behavior and data belong together. A class should expose
clear responsibilities and avoid becoming a container for unrelated methods.

## Collections
{: #collections }

Collections are the vocabulary of everyday Java programs. Choosing the right
collection makes both performance and intent easier to understand.

## Exceptions
{: #exceptions }

Exceptions should communicate failure clearly. Catching broadly or hiding errors
usually makes systems harder to operate.
